# API Usage Guide

1. Start by making a GET request to `/README` (or any other markdown file) to see the available tools in that file's frontmatter.
2. Execute tools by making a POST request to the same markdown file URL with JSON body: `{"command": ["tool", "arg1", "arg2"]}`.
3. You may POST to **any** markdown file that declares tools in its frontmatter; discovered tool pages behave the same way as `/README`.
4. Only commands explicitly listed in a markdown file's frontmatter can be executed. Unlisted commands will return an error.
5. Commands execute relative to the markdown file's directory.
6. Placeholder `{ }` requires exactly ONE argument: `[ls, { }]` accepts `['ls', 'dir']` but rejects `['ls']` or `['ls', 'dir1', 'dir2']`.
7. Placeholder `{ regex: ... }` validates against a pattern: `[cat, { regex: ".*\\.txt$" }]` accepts `['cat', 'file.txt']` but rejects `['cat', 'file.py']`.
8. Commands ending with `;` (semicolon) accept **no** additional flags: `[rm, { }, ;]` cannot accept `['rm', 'file', '-f']`.
9. Commands without `;` accept unlimited flags: `[ls]` can be `['ls', '-la', '--color', '--help']`.
10. Pass environment variables like `$USER` or `$DB` exactly as written. Do **not** substitute values yourself.
11. Missing or invalid placeholder arguments will cause an error.
